# 진척도 확인 워크플로우

## 실행 단계

### 1️⃣ PRD 변경 감지 및 동기화 (최우선)

1. `prd.md` 파일의 마지막 수정 시간 확인
2. PRD 내용과 `progress-checklist.md` 항목 비교
3. **PRD 변경 감지 시**:
   - 새 기능 추가됨 → 체크리스트에 `- [ ]` 항목 추가
   - 기능 삭제됨 → 체크리스트에서 해당 항목 제거
   - 기능 변경됨 → 체크리스트 항목 내용 수정
   - 동기화 완료 후 사용자에게 변경 사항 알림

### 2️⃣ 진척도 확인

4. `progress-checklist.md` 파일을 읽어 현재 체크리스트 상태 확인

5. 실제 코드 디렉토리 구조 확인:
   - `chaldduk-app/app/` - 화면 구성
   - `chaldduk-app/src/` - 컴포넌트, 서비스, 상태 관리
   - `worker/src/` - 백엔드 API

6. 각 체크리스트 항목과 실제 구현 상태 대조:
   - 파일 존재 여부 확인
   - 주요 함수/컴포넌트 구현 여부 확인

7. 완료된 항목은 `[x]`, 진행 중은 `[/]`로 체크리스트 업데이트

### 3️⃣ 리포트 생성

8. 전체 진척률 계산 및 테이블 업데이트

9. 사용자에게 진척도 요약 리포트 제공:
   - PRD 동기화 여부 (변경 사항 있으면 표시)
   - 완료 항목 수 / 전체 항목 수
   - 진척률 (%)
   - 다음 우선순위 작업 추천
